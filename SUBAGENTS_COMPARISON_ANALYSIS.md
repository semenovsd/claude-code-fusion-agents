## Предложения по улучшению существующих наборов

### 1. Гибридный подход к длине промптов

**Проблема:** Длинные промпты обеспечивают качество, но увеличивают потребление токенов.

**Решение:** Создать два уровня агентов:
- **Light версии** (~500-800 токенов) для простых задач
- **Pro версии** (~2000-3000 токенов) для сложных задач

**Пример:**
```yaml
name: backend-developer-light
description: Quick backend tasks
prompt_length: short
use_case: Simple API endpoints, bug fixes

name: backend-developer-pro  
description: Complex backend architecture
prompt_length: detailed
use_case: System design, microservices, performance optimization
```

### 2. Адаптивные промпты

**Идея:** Промпты, которые адаптируются под контекст задачи.

**Реализация:**
- Анализ сложности задачи перед выбором уровня детализации
- Динамическое включение/выключение секций промпта
